# MC8532B API Template

模板 ID：`MC8532B`

匹配规则：

- `device.model_name == "MC8532B"`
- 或 `device.hardware_version` 以 `MC8532B` 开头

当前对应实机：

- 展示名称：`G5 Pro`
- `device_alias_name = "G5 Pro"`
- `device.hardware_version = "MC8532BHW1.0"`

这个模板代表当前后端正式支持的 `G5 Pro` 设备侧接口集合。它与 `MU5250` 的主要差异不在网络和基础识别，而在温度、客户端列表和 Wi-Fi 取数策略上：

- Wi-Fi 主配置继续沿用 `main_2g/main_5g` 自动选择逻辑
- 客户端列表优先读 `/tmp/dhcp.leases`，为空时再回退路由访问列表
- CPU 温度不再依赖 `zwrt_bsp.thermal get_cpu_temp`，而是允许回退到 `thermal_zone` 的 `cpuss-*` sysfs

| snapshot | source |
|----------|--------|
| `net.*` | `zte_nwinfo_api nwinfo_get_netinfo` |
| `net.nrca` / `net.lteca` | `zte_nwinfo_api nwinfo_get_netinfo`（载波聚合描述符，原样透传） |
| `net.wan_status` | `zwrt_router.api router_get_status_no_auth` |
| `wlan.ssid/key/enc` | `wireless.main_2g/main_5g` 自动选择：优先启用中的主 Wi-Fi，回退到第一个存在的主 Wi-Fi |

| `clients.list` | `/tmp/dhcp.leases`；为空时回退 `router_lan_access_list` + `router_wireless_access_list` |
| `sms.unread` | `zwrt_wms_get_wms_capacity` 的 `sms_dev_unread_num + sms_sim_unread_num` |
| `sms.list` | `zte_libwms_get_sms_data {page:0,data_per_page:32,mem_store:1,tags:10,order_by:"order by id desc"}` |
| `nfc.switch` | `zwrt_nfc zwrt_nfc_wifi_get` |
| `traffic.*` | `zwrt_data get_wwandst {"source_module":"deviceui","cid":1,"type":1}` |
| `qos.usb_mode` | `zwrt_bsp.usb list` |
| `qos.qci` / `qos.ambr_*` | 启动时全量扫描 `/data/logfs/key.log` 的 QoS `[DATA]` 日志并缓存 |
| `dhcp.*` | `uci`：`network.lan.ipaddr`、`dhcp.lan.start/limit/leasetime` |
| `system.uptime/mem_*` | `system info` |
| `system.cpu_usage` | `/proc/stat` |
| `system.cpu_temp` | 先试 `zwrt_bsp.thermal` JSON 字段；无有效温度时回退 `thermal_zone` 的 `cpuss-*` / `cpu*` /最高有效温度 |
| `system.model/hostname/fw` | `system board` |

| `device.model_name/hardware_version/alias_name` | `zwrt_zte_mdm.api get_zwrt_common_info` |
| `device.board_name` | `system board` |
| `device.profile` | 基于 `device.model_name` 规范化生成 |
| `device.api_template` | 后端模板选择结果，当前固定为 `MC8532B` |

## Observed Notes

- 当前实机 `get_zwrt_common_info` 没有返回 `device_market_name`，但 `model_name` 和 `device_alias_name` 已足够区分模板和展示名称。
- 当前实机 `zwrt_bsp.thermal get_cpu_temp` / `list` 仅返回 `{"version":"V1.3"}`，不含温度值；因此 sysfs 回退是正式模板的一部分，不是偶发兼容分支。
- 当前实机存在 `cpuss-0..3` thermal zone，温度范围有效，适合作为 CPU 温度来源。
- 当前实机未发现可直接复用的本地屏幕渲染栈（无 `/dev/dri`、无触摸输入、无 DevUI 所需字体目录），因此本模板当前只覆盖 `zwrt-datad` 后端接口适配，不包含 `u60pro-devui` 前端适配。
